# Fusang 首页全粒子头像设计

## 目标

将用户提供的 Fusang 狗狗头像放在首页右下侧，并使用 `QingXia-Ela/Ark-Particle-Imitate` 的图片像素采样、粒子聚合和鼠标扰动思路进行重建。最终画面只显示粒子，不显示原始图片层。

## 视觉

- 头像位于首页首屏右下侧，宽度约为视口的 30% 至 34%，不遮挡品牌标题和代表作入口。
- 粒子使用头像自身的灰白、深色与绿色取样，并在深浅主题中调节亮度与混合模式。
- 初次进入时粒子从轻微散布状态聚合；鼠标靠近时局部排斥，离开后弹性复原。
- 首页雨幕和右上雨窗影像继续保留，头像作为右下侧独立层，带轻微横向视差。
- `prefers-reduced-motion` 下直接显示已聚合的静态粒子，不运行扰动动画。

## 实现

- 新增 React Canvas 组件，加载优化后的头像资源作为离屏采样源，不向 DOM 输出原图。
- 按固定步长采样非透明像素，限制桌面端粒子总数，并在小尺寸横屏下进一步降采样。
- 使用单个 `requestAnimationFrame` 循环执行弹簧回位、阻尼和鼠标排斥；只在画布接近视口时运行。
- 三个循环副本分别持有画布，但仅可见或即将进入视口的副本运行，以保持无缝接缝并控制 CPU。
- 代码保留 MIT 来源说明和第三方版权声明，不引入 lodash 或额外运行时依赖。

## 导航修复

主题按钮的圆点与导航编号使用同一固定首列；“明暗”与其他导航标签使用同一第二列和间距。展开时仅改变标签透明度，不叠加额外横向漂移。

## 自定义鼠标

- 精细指针设备使用约 7px 的绿色 CSS 圆点替代系统鼠标，圆点直接跟随指针。
- 交互圆环使用内联 SVG `circle` 和固定 `viewBox` 绘制，并设置 `vector-effect="non-scaling-stroke"`；缩放动画作用于 SVG 外层，避免大幅缩放时出现锯齿、描边变粗或 low-poly 轮廓。
- 首页左侧导航栏及其全部子元素不触发圆环，只显示普通圆点。
- 首次悬停正文链接、按钮或其他交互元素时，一个大于视口对角线的细圆环缩小至约 58px，并围绕鼠标原点显示。
- 圆环位置以插值缓动跟随，圆点不延迟；从一个交互元素移向另一个交互元素时保持现有圆环，不重复播放缩放动画。
- 离开交互区域后，圆环在约 240ms 内渐隐，并进入 600ms 离去冷却。冷却期间重新进入交互元素时，圆环直接以 58px 淡入，不从大尺寸重新缩小；冷却结束后的下一次进入才重新播放完整收束动画。
- 自定义鼠标层不接收指针事件，不影响链接、画布扰动、横向拖动或文字选择。
- 触摸设备与 `prefers-reduced-motion` 保留系统鼠标。

## 验收

- 首页只看见粒子头像，不存在可见原图层。
- 粒子头像位于右下侧，不遮挡非装饰文字。
- 鼠标接近产生局部扰动，离开后稳定回位。
- 首页首尾循环时头像没有明显闪烁或状态跳变。
- 收起导航保持居中，展开后“明暗”与其他标签对齐。
- 导航区域内只有自定义圆点，不出现交互圆环。
- 圆环 DOM 使用 SVG，最终 58px 尺寸下轮廓连续清晰，收束过程中描边粗细保持稳定。
- 自定义圆点可见；首次进入正文交互元素时圆环从视口外尺度缩入并缓动跟随。
- 交互元素之间移动不重复缩放；离开后圆环渐隐，600ms 冷却内再次进入只淡入固定尺寸圆环。
- 类型检查、单元测试、静态构建和 Chromium 端到端测试通过。
