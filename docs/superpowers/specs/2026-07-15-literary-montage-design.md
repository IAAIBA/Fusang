# Fusang 文学蒙太奇站点设计规范

## 目标

将 `codex.online` 的横向叙事节奏转化为非商业化的个人文学展示站。首页优先引导读者进入代表作，同时提供文章索引、归档和作者介绍。视觉使用夜色、雨、水面与植物影像，深浅主题均以绿色作少量点缀。

## 架构

- Astro 静态生成页面，React island 仅承担首页横向画布。
- GSAP Observer 将滚轮、触控板和拖动映射为连续横向位移。
- 三份同构画布周期形成无缝循环；正文、导航和交互文字保持稳定，装饰层执行视差。
- MDX Content Collection 以 `readingLayout` 管理文章排版；长文、竖排短章、横排短章使用独立阅读布局（避开 Astro MDX 保留字段 `layout`）。
- Vercel 使用纯静态构建，无账户、数据库、订阅、商业模块或运行时函数。

## 路由与导航

- `/`：横向蒙太奇首页。
- `/writing`：文章索引。
- `/archive`：年度归档。
- `/about`：作者说明。
- `/writing/[slug]`：正文。
- 首页导航写入 hash 并定位画布；其他页面进入真实路由。
- 左侧导航默认收窄，hover 或键盘 focus 时展开。

## 视觉与响应式

- 画布约九至十个视口宽，不使用整屏分割线、机械式分页或大面积渐变。
- 主要字体为系统宋体栈，无衬线体仅用于导航、日期与状态文字。
- 移动设备要求横屏；竖屏显示旋转提示。
- `prefers-reduced-motion` 下关闭惯性、循环和视差，保留单周期原生横向阅读。

## 验收

- 首尾连续经过时不可见瞬移。
- 非装饰文字不重叠，三种文章模板均可阅读。
- 深浅主题、hash 深链、导航展开、横屏门控和键盘输入正常。
- Astro 类型检查、单元测试、构建和浏览器验证通过。
